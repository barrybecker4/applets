Introduction and Motivation

 For version 6 of relationship marketing there is a requirement that many of the tables that display database objects be sortable, filterable, searchable, and pageable. We considered several approaches for this:

�        Interactive applet – rejected because of long load time.

�        Interactive javascript object – rejected because it would not scale to millions of rows (and would not be pageable).

�        Use existing java HTMLTable class that is used extensively in contact center. – Rejected for the following reasons. Layout of the table is determined by java implementation and by tags in configuration files. So changes to the table format required knowledge of dna and the java implementation. Current implementation does not scale, even though paging is supported, because all rows are loaded in memory. All rows are returned and then paged through. Filtering is not supported.

�        Create a new HTMLUI component (table) which has java backend and jsp front end – This is the approach we selected because it offered the most flexibility and scalability. The backend will implement an interface and will manage the extraction of the data from the database. It will be scalable because it will only send a single “page” of rows to the client at any time. The format of the UI will be entirely jsp, and hence easy to modify. Predefined filters can be specified in a dna configuration file. We can leverage the expression format and parsing capabilities of the existing expressions framework to achieve filtering.
 

Description of Design 

 Participants in the initial design meeting were Llew Mason, Glenn Cochran, Jeff Brainerd, Barry Becker. We established several things related to the behavior of the UI. Pictures coming soon. For the backend, there will be an interface defined (see below) and a default implement that manages the extraction of rows from a database given the currently viewed page and filter.

Here are a list of the files that I foresee we will need to do the implementation:

modules/htmlui/approot/table/
                      templates/
                         inc_table.jsp
                                (the main include for the table. It will take the following parameters:

                                    height (int) fixed height. If rows don’t fit you get a scrollbar.
                         dlg_filter.jsp    - container for holding the custom filter popup dlg.
                         inc_filter_form.jsp  - include for showing a series of form elements based on columns specified in htmltable.dna.
                      styles/

                     HTMLTableContext.java -loads the htmltable.dna config file and provides interface for accessing its data

modules/<module>/config/<application>/htmltable.dna
         This file will contain the application specific configuration information for the table component. For example. Suppose the module is RM and the application is relmkt, then this file should contain entries for all the business objects that you want to be able to display in a table. Things such as which columns to show, which columns to allow using when filtering, and which filters to allow if any. Suppose the businessObjects that you want to show tables for are IM_INTERACTION, IM_LAUNCH, IM_NODE, RM_AUDIENCE, EMAIL_LINK, EMAIL_TEMPLATE, MESSAGE_TEMPLATE. Then the htmltable.dna configuration file should have entries for each of these things with columns and filters defined. (is this list of BO’s correct?). Here is the possible format of the htmltable.dna file:

DNA {
    “defaultPageSize” Integer “3”,       // this is just a default. You can override for different table specs
    “defaultHeight” Integer “400”,     // can be overridden in jsp
     “tableSpecs” DNA {            // this contains a specification for each bo that you want to show in a table.
          “<boName>” DNA {
               “displayColumns” StringArray [
                      <columns (corresponding to bo properties) to show in a particular order>
               ],
               “filters” DNAArray [     // this is optional, if not present, then no filter dropdown.
                     <list of predefined filters that will appear in a dropdown at the top of the table>
                ]
           },
          :
          “IM_LAUNCH” DNA {
               “displayColumns” StringArray [
                    “label”, “status”, “createDt”, “createdBy”    // property names
               ],
               “filters” DNAArray [    // these filters will appear in a dropdown above the table.
                     {
                          “name” String “all”,      // “all” has special meaning
                           “label” String “allLabel”
                      },
                     {
                           “name” String “active”,
                           “label” String “activeFitlerLabel”,
                           “expression” String “status|isEqualTo|A”
                      },
                     {
                           “name” String “expired”,
                           “label” String “expiredFitlerLabel”,
                           “expression” String “status|isEqualTo|E”
                      },
                     {
                           “name” String “custom”,    // “custom” has special meaning
                           “label” String “customLabel”,
                           “formElements” StringArray [     // the columns to use when building a filter form (in a particular order)
                                “label”, “status”, “createDt”, “createdBy”
                           ],
                           “caseInsensitive” Boolean “false”,   // default could be different per table
                     }
               ]
          }
     }
}
